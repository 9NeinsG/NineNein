CSS


The Syntax is just Setting

Code is extremely Fascist. Complete with an order, hierarchy, and specific orders and calls. Like everything else in our reality; universe; world; and on Earth - CSS is extremely simple and can be very fun and enjoyable to use designing the UI(User Interface) as well as anything visual and behavior. However, mastering CSS can be extremely tricky.

CSS Syntax Sypnosis - More

CSS is not as forgiving as HTML.

Customizing Tags

Class'es

Think of it as teaching the HTML presentation skills.

Variables

Think of them as Classes with different variations/options.

Styles

Think of scribbling notes directly into the HTML.

Unlike HTML : CSS does not have Tags but instead classes-variables-styles.

There are many ways to add CSS but Embed/Inline/Styles will be skipped over.
These are bad practices in general - although they do work.

Instead we will understand the StyleSheet and Definitions.

Each one will start with a "." followed by the Class Name such as "text"
.text
Here we have just created a Class

We must set the Class of .text with the properties of our desired look.
To do this we want to add an opening and closing bracket represented between squigs, curls, braces whatever you want to call them "{}"

.text{}

Just like HTML Tags - what ever is between "{}" will be defined properties.

Lets say we want the .text font to be the size of 18px and black.

.text{font-size : 18px; color : black;}

We can make it more readable which looks better when typed out like this :

.text {
font-size : 18px;
color : black;
}

If we want to use this CSS we go back to HTML and add the Class:
<span class="text">We add the text between the span tag with a class</span>

The Result:

We add the text between the span tag with a class

What if you want a different size font or color or a background?
This is where variables come into the picture.

Lets say we want to keep the size of .text but change the color only.

We already have our .text class.

.text {
font-size : 18px;
color : black;
}

We simply need to create a new Class here. We will create .pb

Can be named whatever you want.

.pb {
color: #0000ff;
}

The new Class .pb has just the {"property" : "color blue";} but we want it to inherit .text for size.

Therefore we will make yet another Class merging both .text and .pb


.text .pb {
color: #0000ff;
}

<span class="text pb"> We add the text between the span tag with Class "text" and "pb" </span>

The result :

We add the text between the span tag with a class

We now have 3 classes

1..text which is 18px and Black.
2..pb which is just Blue and inherited size from body.
3..text .pb which is 18px and Blue.

Add the variable you want to the Class of an HTML tag.

This is CSS syntax - the secret is removing redundancy.

Emmet Syntax


Emmet is a tool to make repetatives simpler. No need to concern yourself with thinking you have to remember everything Emmet. Once you get the hang of the very basics, Emmet becomes self learning as you move into shortcode.

Specially in CSS. Nearly all your Emmet useage will take place in HTML. Where all you generally need to know is which Html tag you would like to use, so lets use the tag <section>.

Instead of using the open tag we will simply type section[Tag].[Dot]pr[Class]{hittab key}. Wahla you just added a full function tag with class using Emmet. [Dot] is the shortcode of class="" whatever follows [Dot] is the Class Variable you added - in this example that would be pr.

Writeing :

section.pr

Is the same as Writeing.

<section class="pr"><section>

Thats the power of Emmet.
