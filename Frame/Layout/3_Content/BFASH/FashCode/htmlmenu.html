Inheriting Hierarchy

This is the Hierarchy of HTML Tags within Tags.
Everytime a Tag Opens - the Tag must be Closed.
Tags can be Opened in Open Tags - as other Tags can be Opened in those Open Tags.

EXAMPLE :

 
        <main class="BlackText">
    <section id="1" class="RedText 12px">

         <section id="2" class="BlueText 16px"></section>

    </section>
</main>

We open a Main tag - Add Section1 open tag - Add Section2 open tag inside Section1 - Add Section2 close tag - Add Section 1 close tag - then we closed the Main tag.

Breakdown
Tag Description
Main The Main tag properties(BlackText) becomes the default properties for everything between the Main open and close tags.
Section1 Section1 will override(RedText) the matching properties(font color) of Main - or add added properties(12px) in Section 1 properties.
Section2 Section2 will override(BlueText , 16px) the matching properties(font color, font size) of both Section1 and the Main - or add added properties in Section2 properties.

Depending on which Tags you are between will determine Order, Appearance, Style, Colors, Sizes, etc.

Every Open Tag must be Closed. Every Tag in a Tag must not cross.

You can not close Main with out first closing the Last open tag(everything within Main) - in this case Section1, however Section1 can not be closed until Section2 is closed.
